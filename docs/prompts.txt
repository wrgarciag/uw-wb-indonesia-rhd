## Task 3 — Refactor `scripts/04_calibration_random_tp.R`

1. **Run the calibration model over 2000–2019** (change `CAL_YEAR_START`/`CAL_YEAR_END` and any downstream window logic so the projection cohort runs `2000` → `2019`; today the file uses 2009–2019). Make sure the input `tps_inpt_part*.rds` and population inputs actually cover 2000 onward; if they don't, flag it rather than silently truncating.
2. **Calibrate against observed GBD prevalent cases and deaths** (GBD "Number" for Prevalence and Deaths, aggregated to 5-year age groups — the existing target-prep logic) and **estimate the structural transition probabilities** (the calibrated IR and CF per location–sex–cause, at the current `GRANULARITY`). Keep the random-search argmin-over-i.i.d.-draws strategy and the "candidate 0 = baseline so calibration is never worse than baseline" guarantee.

   **Death target = ALL-CAUSE deaths, decomposed.** The deaths side of the objective must calibrate to GBD **All-causes** deaths (the "All causes" cause in `dx_include` / `cause_map[["all"]]`), not RHD deaths alone. The all-cause death count for each (location, sex, age-group, year) cell must be reproduced as the sum of (a) the **RHD deaths the model produces** (the disease case-fatality flow, sick→dead via CF) plus (b) **background mortality** (the non-RHD competing mortality, `BG.mx` applied to the surviving population pool). So the model's predicted all-cause deaths = model RHD deaths + background deaths, and *that sum* is what gets compared to the GBD All-causes target. Concretely:
   - Currently the target-prep drops "All causes" (`targets <- gbd[cause != cause_map[["all"]], ...]`) and `combo_error` compares model `Deaths` only to per-cause `gbdDeaths`. Change this so the deaths component of the objective uses the GBD **All-causes** deaths as the target, and the model prediction is the decomposed sum (RHD CF deaths + background mortality deaths) rather than RHD deaths in isolation.
   - Prevalence stays an RHD-specific target (GBD RHD Prevalence vs the model's sick stock), unchanged.
   - Because BG.mx is now pinned by the all-cause deaths fit rather than treated as fixed exogenous input, be explicit about whether background mortality is held fixed and only IR/CF move (RHD deaths + fixed BG must sum to observed all-cause), or whether BG.mx is also a free/derived quantity in the decomposition. Document the choice in the header and keep it consistent with the `enforce_tp_constraints` competing-risk logic (which already treats IR/CF/BG.mx as competing risks summing to ≤ 1).
3. **Validate the estimated probabilities:** after calibration, assert for every row of the calibrated output that IR and CF (and BG.mx) are **between 0 and 1** and **never NA**. Strengthen the existing `stopifnot()` validation block so these are hard failures, and additionally check the competing-risk row sums (IR + BG.mx ≤ 1, CF + BG.mx ≤ 1) as the script already does. If any probability is NA or out of [0,1], `stop()` with a message identifying the offending combo/age.

Keep the existing output contract: `adjusted_searo_part{1..10}.rds`, `calibration_factors_random_tp.csv`, and `calibration_diagnostics_random_tp.csv` in `wd_data`, same schema and chunking. Remember the double-calibration warning: this script bakes the multipliers into IR/CF, so `run_adjustment_model` must stay `FALSE` downstream.
