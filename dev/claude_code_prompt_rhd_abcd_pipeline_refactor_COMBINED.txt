# 1. Required epidemiological model structure

Replace the existing living RHD states:

- `mild`
- `severe`
- `post`

with:

- `A`: minimal or early RHD
- `B`: mild established RHD
- `C`: advanced RHD without clinical complications
- `D`: advanced RHD with complications

The conceptual structure should be:

`No RHD -> A <-> B <-> C -> D -> RHD death`

with competing non-RHD mortality from every living state.

Allow the following annual transitions, controlled from `00_run_all.R`:

- No RHD -> A through incident RHD
- A -> No RHD, optional regression
- A -> B
- B -> A, optional regression
- B -> C
- C -> B, optional regression
- C -> D
- D -> C, optional and defaulting to zero
- A/B/C/D -> RHD death
- A/B/C/D -> other-cause death
- remaining probability -> stay in the same state

Incident RHD must enter stage A directly. Do not add a separate acute rheumatic fever state.

# 2. Surgery must not be a health state

Do not create a surgery, post-surgery, or surgical-recovery state.

Surgery is a clinical intervention or health-service event, not an epidemiological state. Stages C and D and receipt of surgery are not mutually exclusive. A surgery tunnel would also be shorter than the one-year cycle and would risk conflating disease severity with service utilization.

Represent surgery as:

1. a fixed proportion of people in stage C and/or stage D who require surgery during each annual cycle;
2. an annual flow or trace output;
3. an intervention that changes selected transition probabilities;
4. a cost calculated from the surgery flow.

At minimum expose these parameters in `00_run_all.R`:

```r

frac_D_requiring_surgery <- ...

cov_surgery_ref <- ...
cov_surgery_scale_up <- ...

eff_surgery_C_to_D <- ...
eff_surgery_D_to_rhd_death <- ...

cost_surgery <- ...
```

Use clearer names if appropriate, but preserve this conceptual separation.

Suggested annual surgery-need and surgery-volume calculations are:

```r

  C_start_t * frac_C_requiring_surgery

n_D_requiring_surgery_t <-
  D_start_t * frac_D_requiring_surgery

n_surgeries_C_t <-
  n_C_requiring_surgery_t * surgery_coverage_t

n_surgeries_D_t <-
  n_D_requiring_surgery_t * surgery_coverage_t

  n_surgeries_C_t + n_surgeries_D_t
```

If surgery coverage differs by stage, define separate C and D coverage parameters.

Do not automatically remove people from stage C or D after surgery. They remain classified by their epidemiological/clinical RHD stage unless the modeled effect changes their subsequent transition risk.

Model surgery effects through transition-risk reductions:

```r

    1 -
      eff_surgery_C_to_D *
      effective_surgery_reach_C_t

    1 -
      eff_surgery_D_to_rhd_death *
      effective_surgery_reach_D_t
```

where:

```r

```

This is a population-average formulation. If the engine explicitly separates those requiring surgery from those not requiring surgery within the annual flow calculations, document the equivalent weighted-risk formulation.

Do not apply surgery effects to stages A or B.

If there is uncertainty regarding whether surgery reduces C -> D, D -> RHD death, or both, parameterize both effects separately and make either one easy to set to zero.

